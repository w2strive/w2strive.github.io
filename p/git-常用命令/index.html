版本回退

Git 中的版本回退操作说明：

  • HEAD：表示当前版本
  • HEAD^：表示上一个版本
  • HEAD^^：表示上上个版本
  • HEAD~n：表示前 n 个版本（如：HEAD~100）

版本回退操作示例：

# 查看提交历史
$ git log --pretty=oneline
7b5e6e890361d44280dcb7dddbeace9fb8a7db82 (HEAD -> master) append GPL
aed6917ff5d1b1598027509e40bf79d326e25258 add distributed
54fda1ec2c8eee3d8b3d88eb61ba47432cb946c5 wrote a readme file

# 查看命令历史
$ git reflog
7b5e6e8 (HEAD -> master) HEAD@{0}: commit: append GPL
aed6917 HEAD@{1}: commit: add distributed
54fda1e HEAD@{2}: commit (initial): wrote a readme file

# 回退到前两个版本
$ git reset --hard HEAD~2
HEAD 现在位于 54fda1e wrote a readme file

# 恢复到指定版本
$ git reset --hard 7b5e6e8
HEAD 现在位于 7b5e6e8 append GPL

工作区管理

暂存修改

  1. 暂存当前工作：

    • git stash：保存当前工作现场
    • git stash list：查看存储列表
    • git stash pop：恢复并删除第一个存储
    • git stash drop stash_id：删除指定存储

  2. 跨分支修复：

    • 在 master 分支修复 bug 后
    • 使用 git cherry-pick <commit_id> 将修复同步到开发分支

分支管理

分支操作命令

  • git remote -v：查看远程仓库信息
  • git branch -r：查看远程分支列表
  • git branch -a：查看所有分支（本地+远程）
  • git checkout -b branch-name origin/branch-name：创建本地分支并关联远程分支
  • git branch --set-upstream branch-name origin/branch-name：建立分支关联

远程仓库配置

配置远程仓库的标准流程：

# 进入项目目录
cd /path/to/your/git/repo/

# 配置远程仓库
git remote -v                    # 查看当前配置
git remote rm origin            # 删除已有配置
git remote add origin ssh://xxx # 添加新配置

# 推送代码
git push -u origin --all       # 推送所有分支
git push -u origin --tags      # 推送所有标签

高级特性

选择性提取

  • 从其他分支提取指定文件： git checkout <分支名> <文件名或文件夹名>

子模块管理

子模块的基本操作：

# 初始化子模块
git submodule init

# 更新子模块
git submodule update

# 克隆包含子模块的项目
git clone --recurse-submodules <repository-url>
